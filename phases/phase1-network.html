Phase 1: Building the Network Foundation

September 18, 2025 | Author: Santhosh Kumar Raju

In this phase, we establish the networking foundation for our highly available WordPress architecture. This includes creating a VPC, subnets, route tables, and an internet gateway.

Creating the VPC

The Virtual Private Cloud (VPC) serves as the isolated network environment for all our resources. We created a VPC with the following configuration:

  • Name: epicbooks
  • CIDR Block: 10.0.0.0/16
  • Purpose: Provides an isolated virtual network for all project resources

Subnet Configuration

We created both public and private subnets across two availability zones for high availability:

Public Subnets

For resources that need direct internet access (ALB, NAT Gateway, Bastion Hosts):

  • epicbook-public-1 in us-east-1a | CIDR: 10.0.0.0/24
  • epicbook-public-2 in us-east-1b | CIDR: 10.0.1.0/24

Private Subnets

For resources that should not be directly accessible from the internet (EC2 Instances, RDS):

  • epicbook-private-1 in us-east-1a | CIDR: 10.0.2.0/24
  • epicbook-private-2 in us-east-1b | CIDR: 10.0.3.0/24

Internet Gateway

We created and attached an Internet Gateway to allow communication between resources in public subnets and the public internet:

  • Resource: Internet Gateway
  • Name: epic-books-igw
  • Action: Created and attached to the epicbooks VPC

Route Tables Configuration

Public Route Table

  • Name: epicbooks-public-rt
  • Associated Subnets: epicbook-public-1, epicbook-public-2
  • Routes:
    • 10.0.0.0/16 → local (default, for internal VPC traffic)
    • 0.0.0.0/0 → igw-xxxxxxxx (sends all other traffic to the IGW)

Private Route Table

  • Name: epicbooks-private-rt
  • Associated Subnets: epicbook-private-1, epicbook-private-2
  • Routes:
    • 10.0.0.0/16 → local (default, for internal VPC traffic)
    • (Note: A NAT Gateway would be needed here for instances to get software updates)

Summary

This network foundation provides a secure, segmented environment for our WordPress application. The public subnets will host internet-facing resources, while the private subnets will secure our application servers and database. The route tables ensure proper traffic flow between these segments and the internet.
