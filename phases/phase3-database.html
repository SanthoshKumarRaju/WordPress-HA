Phase 3: Database Tier Implementation

September 18, 2025 | Author: Santhosh Kumar Raju

In this phase, we set up a highly available Amazon RDS MySQL instance in private subnets. The database is the backbone of our WordPress application, and we've configured it for both performance and reliability.

RDS Database Configuration

We created an Amazon RDS MySQL instance with the following specifications:

Basic Configuration

  • DB Identifier: wordpress1
  • Engine: MySQL 8.0
  • Instance Class: db.t3.micro (for development)
  • Storage: 20 GB General Purpose SSD (gp2)
  • Multi-AZ Deployment: Enabled for high availability

Network & Security

  • VPC: epicbooks
  • Subnet Group: Custom group with private subnets (epicbook-private-1, epicbook-private-2)
  • Publicly Accessible: No
  • VPC Security Group: db-sg

Database Authentication

  • Master Username: wordpressuser
  • Password: StrongPassword123! (auto-generated and stored in Secrets Manager)
  • Database Name: wordpress

Database Creation Process

We created the WordPress database and user through the MySQL client:

# Connect to the RDS instance from a bastion host or EC2 instance
mysql -h wordpress1.123456789012.us-east-1.rds.amazonaws.com -u admin -p

# Create the WordPress database and user
CREATE DATABASE wordpress;
CREATE USER 'wordpressuser'@'%' IDENTIFIED BY 'StrongPassword123!';
GRANT ALL PRIVILEGES ON wordpress.* TO 'wordpressuser'@'%';
FLUSH PRIVILEGES;
EXIT;

Backup and Maintenance

We configured automated backups and maintenance for the database:

Automated Backups

Enabled with 7-day retention period. Daily snapshots are taken during the preferred backup window.

Multi-AZ Deployment

Synchronous replication to a standby instance in a different Availability Zone for failover support.

Maintenance Window

Configured for weekly maintenance during off-peak hours to minimize application impact.

Performance Considerations

To ensure optimal database performance for WordPress:

  • Enabled Performance Insights for database monitoring
  • Configured parameter group with optimized MySQL settings for WordPress
  • Set up CloudWatch alarms for CPU utilization and storage space
  • Enabled slow query logging to identify performance bottlenecks

Summary

By deploying our MySQL database on Amazon RDS in private subnets, we've created a secure, highly available database tier for our WordPress application. The Multi-AZ deployment ensures database resilience, while the security group configuration restricts access to only the web servers. Automated backups and performance monitoring provide operational excellence and help maintain the health of our database environment.
